# Contributing to AI-DEX

First off, thank you for considering contributing to AI-DEX. It's people like you that make open-source a great community.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue detailing:
- The expected behavior
- The actual behavior
- Steps to reproduce
- Your operating system and browser

### Suggesting Enhancements
We welcome suggestions for new features or improvements. When proposing an enhancement, please include:
- A clear description of the feature
- The problem it solves
- Potential implementation details (if you have them in mind)

### Code Contributions
1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name` or `fix/your-bug-fix`)
3. Make your changes and ensure the code follows the existing style
4. Write clear, descriptive commit messages
5. Push your branch (`git push origin feature/your-feature-name`)
6. Open a Pull Request

## Development Setup
Please refer to the `README.md` for complete instructions on setting up your local environment, including database setup.

## Commit Guidelines
To maintain a clean and readable history, we encourage the following commit style:
- Use present tense ("Add feature" not "Added feature")
- Capitalize the first letter
- Do not end with a period
- Keep subject lines concise

Thank you for contributing!
